# ビジュアル＆テーマ仕様書（サイバーパンク × ミニマル・シンプル）

## 1. ビジュアルコンセプト
*   **「Neo-Minimal Cyber」**
    *   ネオンカラーやグリフ（電子記号）などのサイバーパンク的な記号を最小限・効果的に抽出しつつ、画面全体のレイアウトやUIコンポーネントはフラットで余計な装飾を省いた「ミニマルデザイン」で統一します。
    *   システムが剥き出しになっているような、無機質で機能美あふれる洗練されたパズル画面を目指します。

## 2. 配色（カラーパレット設計）
*   **背景色 (Background)**:
    *   深みのあるダークグレー、または消炭色（例：`#0F172A` / Slate 900 または `#090D16` / 深夜の電子空間を想起させるダークカラー）
*   **メインUI・テキスト (Primary Text/UI)**:
    *   高コントラストなオフホワイト（例：`#F8FAFC`）、または視認性の高いペールブルー（例：`#E2E8F0`）
*   **アクセント（ネオンカラー・警告・成功）**:
    *   **デバッグ・修正成功 / 安全**: ネオングリーン（例：`#10B981` / `#34D399`）
    *   **バグ・エラー / 警告 / 立ち入り禁止エリア**: ネオンレッド、またはホットピンク（例：`#F43F5E` / `#FB7185`）
    *   **プログラム実行中 / 制御対象ギミック**: ネオンシアン、またはエレクトリックブルー（例：`#06B6D4` / `#22D3EE`）

## 3. UI・画面構成方針
*   **視覚的ノイズの徹底排除**:
    *   グラデーションや複雑なテクスチャ、不要な光彩（グロウ）効果は極力使用せず、単色（ソリッド）と極細の線画（1px〜2px）で構成。
    *   枠線や区切り線も最低限にし、十分な余白（ネガティブスペース）を確保してパズル自体に集中できるようにします。
*   **フォントの選定**:
    *   **等幅フォント（Monospace）**: コード入力エリア、ステージ内のオブジェクト情報などに「JetBrains Mono」や「Fira Code」などのモダンな等幅フォントを採用し、プログラミングテーマの統一感を高めます。
    *   **システムUI**: サンセリフ（Interなど）の極細〜中太ウェイトを用いてすっきりと表示します。

## 4. ステージの表現方法
*   **2Dグリッド表現**:
    *   ステージはシンプルに上から見下ろした2Dのマス目（グリッド）で表現します。
    *   **プレイヤーキャラクター**: シンプルな矢印（向きを表す）や、ミニマルなドット/記号。
    *   **壁・障害物**: ソリッドな塗りつぶしのブロック、または細い白枠のブロック。
    *   **ゴール**: 点滅するシアンのゲート、または「`Exit`」や「`0x00`」といったミニマルなテキストグリフ。
    *   **ハッキング対象オブジェクト（高難易度用）**: 制御可能状態のときは水色の枠線、ロックされているときは赤色の枠線で表現。

## 5. アニメーション
*   派手な爆発などの3Dエフェクトは使用せず、2Dのシンプルなトランジション（移動のフェード、テキストのタイピングエフェクト、エラー時の細かな画面のブレ等）で、デジタル的な心地よさを演出します。

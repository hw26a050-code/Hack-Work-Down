# ゲームシステム・モード詳細仕様書

## 1. 使用言語・命令セット（独自簡易命令）
プレイヤーは以下の擬似コード（簡易命令）を用いてステージを攻略します。

### 基本命令（例）
*   `move()`: キャラクターを一歩前進させる
*   `turnLeft()` / `turnRight()`: キャラクターの方向を転換する
*   `if` / `while` などの簡易制御構文（基本・高難易度共通、高難易度ではより複雑なループ・分岐に対応）

## 2. モード構成
ゲームはプレイヤーの習熟度とターゲット層に合わせて、以下の2つのモードで構成されます。

### ① 基本モード（未経験者・初心者向け）
*   **コンセプト**: プログラミングの基礎（順次処理、分岐、反復）を段階的に楽しく学ぶ。
*   **メインシステム**: **既存コードの修正型（イメージA）**
    *   各ステージ開始時に、意図しない動きをする「バグのあるコード」がすでにセットされています。
    *   プレイヤーはキャラクターの初期配置とゴール、およびステージ上の障害物を確認します。
    *   コードの「誤った命令」を書き換えたり、不足している箇所を補ったりして（デバッグ）、正しくゴールに導くことでクリアとなります。

### ② 高難易度モード（プログラミング経験者向け）
*   **コンセプト**: 高度な論理的思考、マルチタスク制御、アルゴリズムの構築。
*   **メインシステム**: **デバッグ ＋ ハッキング・ギミック操作の混在型（イメージA ＋ B）**
    *   基本のデバッグ要素に加え、ステージ内に「ハッキング可能なギミック（セキュリティゲート、移動プラットフォーム、トラップ等）」が存在します。
    *   プレイヤーはキャラクター自身の制御コードだけでなく、ギミックを遠隔制御するサブプログラム（ハッキングコード）も記述・修正する必要があります。
    *   キャラクターの動きとギミックの同期制御など、より高度なタイミングや論理分岐が要求されます。
